# Возможности и маршруты

## Guest mode

Точка входа: `GuestApp.tsx`. Все данные хранятся в **localStorage**; сервер не требуется.

Guest-сборку можно **установить как PWA** (Add to Home Screen / «Установить приложение»). После первой загрузки статика кэшируется service worker — приложение открывается офлайн, локальные тесты и прогресс в localStorage доступны без сети. При новом деплое показывается баннер «Доступна новая версия».

### Маршруты

| Маршрут | Страница | Описание |
|---------|----------|----------|
| `/` | Главная | Обзор возможностей, ссылки на разделы |
| `/tests` | Локальные тесты | Список тестов из localStorage: пагинация, сортировка, удаление, кнопка «Пройти» |
| `/editor` | Редактор | Создание нового теста |
| `/editor/:id` | Редактор | Редактирование существующего локального теста |
| `/import` | Импорт | Загрузка JSON из файла или вставка из буфера |
| `/info` | Инфо | Формат JSON и генератор промпта для ИИ |
| `/play/:id` | Прохождение | Интерактивное прохождение локального теста |

### Автосид демо-тестов

При первом запуске guest-режима, если localStorage пуст, автоматически создаётся **11 демо-тестов** по темам (C#, JavaScript, TypeScript, Python, Go, React, PostgreSQL, ClickHouse, Docker, Kubernetes, Linux). Логика: `seedLocalTests.ts` → `seedTests.ts` + JSON в `src/guest/data/seed/questions/`.

### Импорт JSON

Страница `/import` принимает JSON в формате `TestDefinition` и **создаёт новый** локальный тест целиком. Валидация в `import.ts`:

- корректный JSON-объект;
- обязательное непустое поле `name`;
- минимум один вопрос;
- у каждого вопроса — непустой `text`, минимум 2 ответа;
- поле `correct` — индекс правильного ответа (с нуля), в пределах массива `answers`;
- необязательное поле `explanation` — пояснение к правильному ответу (показывается после ответа в TestPlayer; настройки видимости — в localStorage).

Функция `parseQuestionsJson` принимает тот же формат вопросов, но **не требует** `name` — достаточно `{ questions: [...] }` или полного теста (берётся только массив `questions`).

При ошибке выбрасывается `ImportValidationError` с понятным сообщением.

### Экспорт JSON

Из редактора доступен экспорт через `export.ts`:

- `exportTestToJson` — сериализация в строку;
- `downloadTestJson` — скачивание файла `.json`.

Экспортируются поля `name`, `description`, `questions` (с `text`, `answers`, `correct`, опционально `explanation`).

### Страница «Инфо» (`/info`)

`InfoPage.tsx` — две вкладки:

- **Формат JSON** — правила схемы и готовый шаблон из `importTemplate.ts`; кнопка копирует шаблон в буфер обмена.
- **Генерировать промт** — форма (тема, количество вопросов и ответов, сложность, пояснения); кнопка копирует промпт для ИИ в буфер обмена. Прямая ссылка: `/info?tab=prompt`.

## Full mode

Точка входа: `FullApp.tsx`. Включает всё из guest для **локальных** тестов плюс облачные возможности через API.

### Маршруты

| Маршрут | Доступ | Описание |
|---------|--------|----------|
| `/` | все | Главная |
| `/login` | все | Вход |
| `/register` | все | Регистрация |
| `/tests` | все | Публичный каталог тестов (пагинация, сортировка) |
| `/tests/:id` | все | Карточка публичного теста |
| `/tests/:id/play` | авторизован | Прохождение с сохранением на сервере |
| `/editor` | все | Новый **локальный** тест (как в guest) |
| `/editor?storage=cloud` | авторизован | Новый облачный тест |
| `/editor/:id` | все | Редактирование (UUID → local, числовой id → api) |
| `/import` | все | Импорт JSON в localStorage |
| `/info` | все | Формат JSON и генератор промпта для ИИ |
| `/my/tests` | все | Мои тесты: вкладки **Локальные** (localStorage) и **Облачные** (CRUD, публикация, обмен с локальным хранилищем; требует входа для облака) |
| `/play/:id` | все | Прохождение локального теста |
| `/profile` | авторизован | Статистика и история прохождений |
| `/applications` | авторизован | Заявки: создание ссылки на непубличный тест для участника |
| `/application/:token` | все | Прохождение теста по ссылке заявки (без входа) |

Защищённые маршруты обёрнуты в `ProtectedRoute` (редирект на `/login`). Маршрут `/local/tests` перенаправляет на `/my/tests`.

### Облачные возможности (авторизованный пользователь)

- **Создание** теста на сервере (изначально приватный)
- **Публикация / снятие с публикации** (`PUT .../publish`, `PUT .../unpublish`)
- **Дополнение** теста новыми вопросами без удаления старых (`PATCH /api/tests/{id}`)
- **Изменение** названия и описания (`PATCH /api/tests/{id}/info`)
- **Обмен cloud ↔ local** — выгрузка облачного теста в JSON или localStorage (`GET /api/tests/{id}/full`); загрузка локального теста в облако как нового (`POST /api/tests`)
- **Удаление** теста (`DELETE /api/tests/{id}`)
- **Прохождение** публичного теста с записью ответов на сервер
- **Пересдача** — удаление результата и повторное прохождение
- **Дорешивание** — при добавлении вопросов к уже начатому тесту
- **Профиль** — три блока статистики (мои тесты; прохождение своих; прохождение чужих) и история
- **Заявки** — персональная ссылка на прохождение **непубличного** теста без регистрации участника; см. раздел [Заявки](#заявки)

## Заявки

Заявка — способ дать конкретному человеку пройти **приватный** (неопубликованный) облачный тест. Есть два типа:

| Тип | Описание | Ссылка автору | Прохождение |
|-----|----------|---------------|-------------|
| **По ссылке** (`link`, по умолчанию) | Универсальная ссылка для любого участника | Да, копирование | Анонимно по токену |
| **Для внутреннего пользователя** (`internalUser`) | Персональное предложение зарегистрированному пользователю | Нет | Только назначенный пользователь с JWT |

У каждой заявки есть обязательное поле **название** (`title`) — подпись в списках автора и получателя, не связанная с именем участника.

### Кто и что делает

| Роль | Действие |
|------|----------|
| **Автор** (владелец теста) | На странице `/applications` создаёт заявку: указывает **название**, выбирает тип (**по ссылке** или **для внутреннего пользователя**), при типе internal — **выбирает пользователя системы**, выбирает непубличный тест, при необходимости включает «Скрыть результат от участника». Для типа link получает ссылку `/application/{token}`; для internal — подтверждение отправки без URL. |
| **Участник (link)** | Открывает ссылку без входа и проходит тест в `TestPlayer`. |
| **Получатель (internal)** | Видит предложение во вкладке **«Мне предложено»** на `/applications?tab=incoming` и проходит тест (JWT обязателен). Анонимный доступ к internal-заявке по токену возвращает **403**. |

Создавать заявки можно **только для непубличных** тестов, принадлежащих текущему пользователю. Для опубликованных тестов используется обычное прохождение через `/tests/:id/play`.

### Страница автора (`/applications`)

`ApplicationsPage.tsx`:

- вкладки **«Мои заявки»** и **«Мне предложено»** (`?tab=incoming`);
- форма **«Новая заявка»**: **название заявки**, переключатель типа (**по ссылке** / **для внутреннего пользователя**), при internal — кнопка «Выбрать пользователя» (поиск по имени/email), выбор теста из списка приватных, чекбокс **«Скрыть результат от участника»**;
- после создания link-заявки — готовая ссылка и кнопка «Копировать»; после internal — сообщение об отправке пользователю;
- список **«Мои заявки»** с пагинацией: название, тест, метка типа (**по ссылке** / **внутренняя**), дата, статус (пройден / не пройден), балл (для автора всегда виден), метка **«без результата»** при включённом флаге скрытия; кнопка **«Копировать ссылку»** только для типа link;
- вкладка **«Мне предложено»**: название, автор, тест, дата, статус и кнопка **«Пройти»** → `/application/{token}`.

Автор видит итог участника (`correctAnswers`, `scorePercent`) в списке заявок независимо от того, скрыт ли результат от участника.

### Прохождение по ссылке (`/application/:token`)

`ApplicationPlayPage.tsx` передаёт в `TestPlayer` источник `{ type: 'application', token }`.

Для заявок типа **link** участник **не авторизуется**. Для **internal** — требуется вход под назначенным пользователем; иначе API возвращает 403 «Доступ к этой заявке только для назначенного пользователя».

Прогресс сессии дублируется в localStorage (`vibetest_progress_application_{token}`) для удобства навигации между вопросами, но **перепрохождение запрещено на сервере**: после ответа на все вопросы заявка получает `CompletedAt`, повторные `submit` возвращают ошибку «Тест по этой заявке уже пройден».

При повторном открытии уже завершённой ссылки сразу показывается экран завершения (без кнопки «Пройти снова»).

### Режим «Скрыть результат от участника»

Если при создании заявки включён флаг `hideResultsFromParticipant`:

| Аспект | Поведение для участника |
|--------|-------------------------|
| Ответ на вопрос | Ответ сохраняется, но **нет** подсветки правильного/неправильного и **нет** пояснений (`explanation`) |
| Навигация по вопросам | Точки только «отвечен» / «не отвечен» (нейтральный стиль) |
| Выбранный вариант | Отображается нейтрально, без оценки |
| Итог | Экран «Тест завершён. Спасибо!» **без** процента и счётчика правильных ответов |
| `GET .../result` | **403 Forbidden** — участник не может получить балл через API |

Для автора заявки результаты по-прежнему доступны в списке заявок.

### Отличия от обычного API-прохождения

| | Публичный тест (`type: 'api'`) | Заявка link (`type: 'application'`) | Заявка internal |
|--|-------------------------------|-------------------------------------|-----------------|
| Авторизация | JWT обязателен | Не нужна | JWT назначенного пользователя |
| Тип теста | Опубликованный | Приватный | Приватный |
| Пересдача | Да (`DELETE .../result`) | **Нет** | **Нет** |
| Обратная связь после ответа | Полная (если не отключена в настройках) | Полная или нейтральная (флаг скрытия) | Полная или нейтральная (флаг скрытия) |
| Кнопка «Пройти снова» | Есть | **Нет** | **Нет** |

Подробнее об эндпоинтах: [api-reference.md § Заявки](api-reference.md#заявки).

### Списки тестов

На страницах `/tests`, `/my/tests` (вкладка «Локальные») и `/my/tests` (вкладка «Облачные») над списком отображается общий тулбар `TestListToolbar`:

- счётчик «Всего: N»;
- выбор размера страницы (10 / 20 / 50 / 100);
- сортировка: по имени (А→Я / Я→А) или по дате обновления (сначала новые / старые; по умолчанию — недавно обновлённые).

Локальный список сортируется на клиенте; облачные списки — через query-параметры `sortBy` и `order` API.

На вкладке **Облачные** (`MyTestsPage`): кнопки «Скачать JSON» и «Сохранить локально» (через `GET /full` и `testTransfer.ts`).

На вкладке **Локальные** в full-режиме: «Скачать JSON», «В облако» (`POST /api/tests`). В guest-режиме (`LocalTestsPage`) — только «Скачать JSON».

## TestEditor

Общий компонент `TestEditor` с пропом `mode: 'local' | 'api'`.

| Режим | Загрузка | Сохранение |
|-------|----------|------------|
| **local** | `getLocalTestById` из localStorage | `saveLocalTest` / `createLocalTestFromDefinition` |
| **api** (новый) | пустая форма | `POST /api/tests` |
| **api** (редактирование) | `GET /api/tests/{id}/full` (с правильными ответами) | `PATCH .../info` + `PATCH ...` (дополнение вопросов) |

Особенности API-режима:

- Существующие вопросы отображаются как **заблокированные** (`lockedQuestions`) — их нельзя удалить или изменить (иммутабельность на сервере).
- Новые вопросы добавляются в отдельный блок и отправляются через `appendQuestions`.
- Доступны фазы: редактирование → предпросмотр → сохранение.
- Валидация перед сохранением использует те же правила, что и импорт JSON.
- **Экспорт JSON** — в режиме `local` и при редактировании облачного теста (`mode=api`, `isEdit`): полный `TestDefinition` из заблокированных и новых вопросов.
- **Пояснение к ответу** — необязательное поле `explanation` у вопроса; по умолчанию показывается после любого ответа; в TestPlayer настройки «при правильном / при неправильном» хранятся в `vibetest_player_explanation_settings`.
- **Импорт в редакторе** (`TestImportPanel`): раскрывающийся блок «Импорт вопросов из JSON» после описания дополняет текущий тест вопросами из JSON (файл или вставка). Поля `name` и `description` не меняются; при одном пустом стартовом вопросе он удаляется перед добавлением. В отличие от `/import`, тест не создаётся заново — вопросы добавляются к уже открытому.

В guest `EditorPage` всегда передаёт `mode="local"`. В full `/editor` по умолчанию — `mode="local"`; облачное создание — `/editor?storage=cloud`. При `/editor/:id` с **числовым** id — `mode="api"`; с **нечисловым** id (UUID) — `mode="local"`.

## TestPlayer

Общий компонент `TestPlayer` с источником `source`:

```typescript
{ type: 'local'; testId: string }   // guest и локальные тесты в full
{ type: 'api'; testId: number }     // публичные тесты full-режима
{ type: 'application'; token: string }  // прохождение по ссылке заявки
```

Опционально для `application` в источнике можно передать `hideResults` и `isCompleted` (обычно загружаются из `GET /api/applications/{token}`).

### Фазы прохождения

`loading` → `playing` → `finished` (или ошибка).

### Локальный режим (`type: 'local'`)

- Тест читается из localStorage.
- Ответы проверяются на клиенте (известны правильные варианты).
- Прогресс сохраняется в `vibetest_progress_{id}`.
- Результат сохраняется в `vibetest_guest_results`.
- Можно переключаться между вопросами, видеть индикаторы правильности.
- После ответа под вариантами может показываться `explanation` (если задано и не отключено в настройках).
- Настройки пояснений: `vibetest_player_explanation_settings` в localStorage (`showOnCorrect`, `showOnIncorrect`, по умолчанию `true`).

### API-режим (`type: 'api'`)

- Вопросы загружаются через `GET /api/tests/{id}` (без правильных ответов и без пояснений).
- Каждый ответ отправляется на `POST /api/tests/{id}/submit`; ответ может содержать `explanation`, если оно задано у вопроса.
- Прогресс дублируется в localStorage (`vibetest_progress_api_{id}`) для восстановления сессии.
- Итог — `GET /api/tests/{id}/result`; поддерживается дорешивание и пересдача.

### Режим заявки (`type: 'application'`)

- Данные загружаются через `GET /api/applications/{token}` (`ApplicationPlayResponse`: тест, `title`, `hideResultsFromParticipant`, `isCompleted`).
- Ответы отправляются на `POST /api/applications/{token}/submit`.
- Прогресс в localStorage: `vibetest_progress_application_{token}`.
- **Пересдача недоступна** — после завершения всех вопросов повторный submit отклоняется сервером.
- При `hideResultsFromParticipant` участник не видит оценку и не вызывает `GET .../result`; итог — экран `TestResultSummary` с `variant: 'submitted'`.
- При повторном заходе на завершённую заявку сразу показывается экран завершения.

### Восстановление прогресса

При повторном открытии теста `TestPlayer` читает сохранённый прогресс из localStorage и продолжает с последнего вопроса.

## Сводка: guest vs full

| Функция | Guest | Full |
|---------|-------|------|
| Локальные тесты | `/tests` | `/my/tests` (вкладка «Локальные») |
| Публичный каталог | — | `/tests` |
| Прохождение | `/play/:id` | `/play/:id` (локальные) + `/tests/:id/play` (API) |
| Регистрация / вход | — | `/register`, `/login` |
| Облачный редактор | — | `/editor` + `/my/tests` |
| Копия теста | — | через локальную выгрузку и загрузку |
| Профиль / статистика | — | `/profile` |
| Заявки (ссылка на приватный тест) | — | `/applications`, `/application/:token` |
| Импорт / инфо | `/import`, `/info` | те же маршруты |
| Seed 11 тестов | да (при пустом storage) | да (при пустом storage, локальная вкладка) |

Подробнее об API: [api-reference.md](api-reference.md) · [spec.md §6](spec.md#6-api-эндпоинты)
