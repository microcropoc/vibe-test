# Возможности и маршруты

## Guest mode

Точка входа: `GuestApp.tsx`. Все данные хранятся в **localStorage**; сервер не требуется.

Guest-сборку можно **установить как PWA** (Add to Home Screen / «Установить приложение»). После первой загрузки статика кэшируется service worker — приложение открывается офлайн, локальные тесты и прогресс в localStorage доступны без сети. При новом деплое показывается баннер «Доступна новая версия».

### Маршруты

| Маршрут | Страница | Описание |
|---------|----------|----------|
| `/` | Главная | Обзор возможностей, ссылки на разделы |
| `/tests` | Локальные тесты | Список тестов из localStorage: пагинация, сортировка, удаление, кнопка «Пройти» |
| `/editor` | Редактор | Создание нового теста |
| `/editor/:id` | Редактор | Редактирование существующего локального теста |
| `/import` | Импорт | Загрузка JSON из файла или вставка из буфера |
| `/info` | Формат JSON | Описание схемы, пример шаблона, копирование в буфер |
| `/play/:id` | Прохождение | Интерактивное прохождение локального теста |

### Автосид демо-тестов

При первом запуске guest-режима, если localStorage пуст, автоматически создаётся **11 демо-тестов** по темам (C#, JavaScript, TypeScript, Python, Go, React, PostgreSQL, ClickHouse, Docker, Kubernetes, Linux). Логика: `seedLocalTests.ts` → `seedTests.ts` + JSON в `src/guest/data/seed/questions/`.

### Импорт JSON

Страница `/import` принимает JSON в формате `TestDefinition` и **создаёт новый** локальный тест целиком. Валидация в `import.ts`:

- корректный JSON-объект;
- обязательное непустое поле `name`;
- минимум один вопрос;
- у каждого вопроса — непустой `text`, минимум 2 ответа;
- поле `correct` — индекс правильного ответа (с нуля), в пределах массива `answers`.

Функция `parseQuestionsJson` принимает тот же формат вопросов, но **не требует** `name` — достаточно `{ questions: [...] }` или полного теста (берётся только массив `questions`).

При ошибке выбрасывается `ImportValidationError` с понятным сообщением.

### Экспорт JSON

Из редактора доступен экспорт через `export.ts`:

- `exportTestToJson` — сериализация в строку;
- `downloadTestJson` — скачивание файла `.json`.

Экспортируются поля `name`, `description`, `questions` (с `text`, `answers`, `correct`).

### Страница «Инфо» (`/info`)

`InfoPage.tsx` показывает правила формата JSON и готовый шаблон из `importTemplate.ts`. Кнопка копирует шаблон в буфер обмена для быстрого старта импорта.

## Full mode

Точка входа: `FullApp.tsx`. Включает всё из guest для **локальных** тестов плюс облачные возможности через API.

### Маршруты

| Маршрут | Доступ | Описание |
|---------|--------|----------|
| `/` | все | Главная |
| `/login` | все | Вход |
| `/register` | все | Регистрация |
| `/tests` | все | Публичный каталог тестов (пагинация, сортировка) |
| `/tests/:id` | все | Карточка публичного теста |
| `/tests/:id/play` | авторизован | Прохождение с сохранением на сервере |
| `/editor` | все | Новый тест (локальный или облачный — см. редактор) |
| `/editor/:id` | все | Редактирование (локальный id или серверный numeric id) |
| `/import` | все | Импорт JSON в localStorage |
| `/info` | все | Формат JSON (как в guest) |
| `/local/tests` | все | Локальные тесты (аналог guest `/tests`) |
| `/my/tests` | авторизован | Мои облачные тесты: CRUD, публикация, fork |
| `/profile` | авторизован | Статистика и история прохождений |

Защищённые маршруты обёрнуты в `ProtectedRoute` (редирект на `/login`).

### Облачные возможности (авторизованный пользователь)

- **Создание** теста на сервере (изначально приватный)
- **Публикация / снятие с публикации** (`PUT .../publish`, `PUT .../unpublish`)
- **Дополнение** теста новыми вопросами без удаления старых (`PATCH /api/tests/{id}`)
- **Изменение** названия и описания (`PATCH /api/tests/{id}/info`)
- **Fork** — копия своего теста (`POST /api/tests/{id}/fork`)
- **Удаление** теста (`DELETE /api/tests/{id}`)
- **Прохождение** публичного теста с записью ответов на сервер
- **Пересдача** — удаление результата и повторное прохождение
- **Дорешивание** — при добавлении вопросов к уже начатому тесту
- **Профиль** — статистика (создано / опубликовано / пройдено / средний балл) и история

---

### Списки тестов

На страницах `/tests`, `/local/tests`, `/my/tests` над списком отображается общий тулбар `TestListToolbar`:

- счётчик «Всего: N»;
- выбор размера страницы (10 / 20 / 50 / 100);
- сортировка: по имени (А→Я / Я→А) или по дате обновления (сначала новые / старые; по умолчанию — недавно обновлённые).

Локальный список сортируется на клиенте; облачные списки — через query-параметры `sortBy` и `order` API.

## TestEditor

Общий компонент `TestEditor` с пропом `mode: 'local' | 'api'`.

| Режим | Загрузка | Сохранение |
|-------|----------|------------|
| **local** | `getLocalTestById` из localStorage | `saveLocalTest` / `createLocalTestFromDefinition` |
| **api** (новый) | пустая форма | `POST /api/tests` |
| **api** (редактирование) | `GET /api/tests/{id}/full` (с правильными ответами) | `PATCH .../info` + `PATCH ...` (дополнение вопросов) |

Особенности API-режима:

- Существующие вопросы отображаются как **заблокированные** (`lockedQuestions`) — их нельзя удалить или изменить (иммутабельность на сервере).
- Новые вопросы добавляются в отдельный блок и отправляются через `appendQuestions`.
- Доступны фазы: редактирование → предпросмотр → сохранение.
- Валидация перед сохранением использует те же правила, что и импорт JSON.
- **Импорт в редакторе** (`TestImportPanel`): раскрывающийся блок «Импорт вопросов из JSON» после описания дополняет текущий тест вопросами из JSON (файл или вставка). Поля `name` и `description` не меняются; при одном пустом стартовом вопросе он удаляется перед добавлением. В отличие от `/import`, тест не создаётся заново — вопросы добавляются к уже открытому.

В guest `EditorPage` всегда передаёт `mode="local"`; в full — `mode="api"` при работе с облачными тестами (маршрут `/editor/:id` с числовым id).

## TestPlayer

Общий компонент `TestPlayer` с источником `source`:

```typescript
{ type: 'local'; testId: string }   // guest и локальные тесты в full
{ type: 'api'; testId: number }     // публичные тесты full-режима
```

### Фазы прохождения

`loading` → `playing` → `finished` (или ошибка).

### Локальный режим (`type: 'local'`)

- Тест читается из localStorage.
- Ответы проверяются на клиенте (известны правильные варианты).
- Прогресс сохраняется в `vibetest_progress_{id}`.
- Результат сохраняется в `vibetest_guest_results`.
- Можно переключаться между вопросами, видеть индикаторы правильности.

### API-режим (`type: 'api'`)

- Вопросы загружаются через `GET /api/tests/{id}` (без правильных ответов).
- Каждый ответ отправляется на `POST /api/tests/{id}/submit`.
- Прогресс дублируется в localStorage (`vibetest_progress_api_{id}`) для восстановления сессии.
- Итог — `GET /api/tests/{id}/result`; поддерживается дорешивание и пересдача.

### Восстановление прогресса

При повторном открытии теста `TestPlayer` читает сохранённый прогресс из localStorage и продолжает с последнего вопроса.

## Сводка: guest vs full

| Функция | Guest | Full |
|---------|-------|------|
| Локальные тесты | `/tests` | `/local/tests` |
| Публичный каталог | — | `/tests` |
| Прохождение | `/play/:id` | `/tests/:id/play` (API) + локальное |
| Регистрация / вход | — | `/register`, `/login` |
| Облачный редактор | — | `/editor` + `/my/tests` |
| Импорт / инфо | `/import`, `/info` | те же маршруты |
| Seed 11 тестов | да (при пустом storage) | нет (только в `isGuestMode`) |
